# Template: ICT Third-Party Register of Information

Supports DORA Art. 28(3). Keep a register of all your contracts for ICT services, marking clearly which ones support critical or important functions. You must report this register to your competent authority at least once a year.

Line this up with the official ITS on the register of information. The ESAs publish a detailed, structured template (several linked tables covering the entity, the providers, the contracts, the functions, and the assessments). The fields below are a practical working subset. Confirm and expand them against the current ITS before you submit to a regulator.

## Core fields (one row per contract)

| Field | Description |
|---|---|
| Arrangement ID | Unique reference |
| Provider legal name | ICT third-party service provider |
| Provider identifier | LEI or equivalent |
| Provider country | Country of establishment |
| Service description | What the ICT service is |
| Service type | For example cloud, software, data centre, network, managed service |
| Supports critical or important function? | Yes / No |
| Function(s) supported | Linked business function(s) |
| Contract start and end | Dates |
| Notice period | For termination |
| Data locations | Where data is stored or processed |
| Sub-outsourcing? | Yes / No; if yes, the chain and countries |
| Substitutability | Easy / Difficult / Not substitutable |
| Concentration flag | Multiple critical functions on this provider? |
| Audit and access rights secured? | Yes / No |
| TLPT cooperation clause (if critical or important)? | Yes / No / N/A |
| Exit strategy documented? | Yes / No |
| Exit strategy tested? | Yes / No / date |
| CTPP designated? | Yes / No / Unknown |
| Risk rating | Critical / High / Medium / Low |
| Contract owner | Accountable owner |
| Last due diligence | Date |
| Last review | Date |

## Maintenance checklist

- Reconcile against accounts-payable and cloud or SaaS inventories to check the register is complete.
- Include intra-group ICT arrangements.
- Flag and monitor concentration (one provider behind many critical functions).
- Track changes to sub-outsourcing and exposure to third countries.
- Confirm the yearly submission to the competent authority is scheduled.
